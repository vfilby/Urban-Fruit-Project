# API Specification

## Design Principles

- REST API, JSON payloads
- All endpoints prefixed with `/api/v1/`
- Authentication via Bearer token (JWT)
- Public endpoints (browsing, viewing) require no auth
- Write endpoints require authentication
- Standard HTTP status codes
- Pagination on list endpoints (cursor-based)

## Endpoints

### Auth

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| POST | `/api/v1/auth/register` | no | Create account |
| POST | `/api/v1/auth/login` | no | Get JWT token |
| POST | `/api/v1/auth/logout` | yes | Invalidate token |
| GET | `/api/v1/auth/me` | yes | Current user profile |
| PATCH | `/api/v1/auth/me` | yes | Update profile |
| POST | `/api/v1/auth/change-password` | yes | Change password |

### Plant Locations

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/api/v1/locations` | no | List/search locations |
| POST | `/api/v1/locations` | yes | Create a location |
| GET | `/api/v1/locations/:id` | no | Get location details |
| PATCH | `/api/v1/locations/:id` | yes | Update location (owner only) |
| DELETE | `/api/v1/locations/:id` | yes | Soft-delete (owner only) |

**Query parameters for GET /locations:**
- `bbox=sw_lat,sw_lng,ne_lat,ne_lng` — bounding box filter
- `species=<id>` — filter by species
- `status=active,unverified` — filter by status (comma-separated)
- `q=<text>` — text search (name, description)
- `near=lat,lng&radius=5000` — proximity search (radius in meters)
- `cursor=<token>` — pagination cursor
- `limit=50` — results per page (max 100)

### Species

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/api/v1/species` | no | List all species |
| GET | `/api/v1/species/:id` | no | Get species detail |

Species are admin-managed in V1 (seeded data). No create/update endpoints for users.

### Photos

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/api/v1/locations/:id/photos` | no | List photos for location |
| POST | `/api/v1/locations/:id/photos` | yes | Upload photo |
| DELETE | `/api/v1/photos/:id` | yes | Delete photo (uploader only) |

Photo upload uses `multipart/form-data`. Response includes S3 URLs for thumbnail
and full-size image.

### Notes

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/api/v1/locations/:id/notes` | no | List notes for location |
| POST | `/api/v1/locations/:id/notes` | yes | Add a note |
| DELETE | `/api/v1/notes/:id` | yes | Delete note (author only) |

## Error Response Format

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Human-readable description",
    "details": [
      {"field": "email", "message": "is already taken"}
    ]
  }
}
```

## Spatial Query Notes

The `bbox` and `near` parameters use PostGIS for efficient spatial queries.
The database should use a spatial index on the (latitude, longitude) columns.
Consider using PostGIS `geography` type for accurate distance calculations.
